## Philosophy

Every document in this repository has a single responsibility and a defined audience. No document duplicates another. If information appears in two places, one of them is the source of truth and the other should link to it.

---

## Closing

Documentation is a living part of the project. As the architecture evolves, the documents that describe it must evolve with it. When you make a change — to the engine, to a convention, to a public interface — update the relevant documents before closing the pull request.
